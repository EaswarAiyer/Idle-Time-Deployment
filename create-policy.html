Autonomous Deployment — Intelligent, idle-aware patching

The system continuously monitors user activity and automatically handles patch delivery and installation with zero manual scheduling. Non-disruptive patches deploy silently at any time; high-impact patches wait for the user's idle window — protecting productivity while keeping machines fully patched.

How it works

📦
Staged Download
Patches downloaded in the background before idle time
🕐
Idle Detection
System detects sustained user inactivity
🔔
Prior Notification
User notified 5 min before deployment starts
⚡
Silent Install
Patches install while the machine is idle

Non-Impact Patches

Install silently, no user disruption

Auto-install anytime
✓
Installs at any time — does not require the machine to be idle. Deployment proceeds during normal working hours without interrupting the user.
✓
No reboot & no app kill — patches complete without closing applications or disrupting running processes.
↓
Pre-staged downloads — binaries fetched in the background well ahead of installation so no bandwidth spike disrupts the user.
✓
Zero productivity impact — the user never sees a prompt or slowdown during installation.
⚠️

High-Impact Patches

Require app kill or immediate reboot

Idle-time only
⏸
Waits for user idle time — installation is held back until the system detects the user is inactive, ensuring no running work is interrupted.
⚠️
App kill & reboot handled automatically — any required application terminations and reboots happen during the idle window, not during work hours.
↓
Pre-staged downloads — patch binaries are ready before the idle window opens so installation begins immediately, minimising the idle time needed.
🔔
5-minute advance notification — if the user returns during idle deployment, a notification gives them the option to postpone before any disruptive action begins.
🤖
AI Agent awareness — long-running AI processes receive a graceful shutdown signal before any force-kill, preserving context and preventing data loss.

Notification & Idle Settings

Pre-deployment notification
A notification toast is shown to the user this many minutes before an idle-time deployment begins, allowing them to postpone if needed.
Allow user to postpone
When the pre-deployment notification appears, allow the user to defer the installation to the next idle window.
Force deployment after
Even if the user keeps postponing, deployment will be forced once this many days have passed since the patch became available.
days
Idle time threshold
Minimum duration of user inactivity before the system is considered idle and eligible for high-impact patch deployment.

App Kill Behavior

Application kill mode
Controls how running applications are handled when a high-impact patch needs to install during idle time.
Graceful shutdown window
After signalling an app to close, wait this long for it to exit cleanly before force-killing it.
⚠️
All running applications will be force-killed without warning as soon as the idle window opens. Unsaved work in those apps will be lost. Use only when strict compliance deadlines require it.
AI Agent awareness
Detect known AI agent and long-running autonomous processes. These receive a graceful shutdown signal and an extended grace period before any force-kill, preventing context loss.
